数据说明文档

基础说明

  • 每周一是一周的开始；

  • 统计中剔除公司内部人员消费数据；

  • 柱形图：量价额等数据；

  • 线形图：增长率等数据；

  • 堆积柱形图：占比类数据；

  • 部分图表的呈现方式仍有待调整。

集团数据

部分数据说明：

  1. 过去四周收入周环比(%):

    示例：（2月18日到3月11日4周收入）/（2月11日到3月4日4周收入）-1
    目的：如果计算单周环比，%数值波动很大；算法通过引入公共周进行平滑。

  2. 过去四周收入月环比(%):

    示例：（2月18日到3月11日4周收入）/（1月21日到2月11日4周收入）-1
    结果：与月度数据环比结果接近。

遗留问题：

  • Flipos导出的餐饮收入包含有场景产品的销售，需要剔除。

场景数据

部分数据说明：

  1. 会员分析 - 留存和流失 - 一个月留存率:

    示例：上个月100个新增客户中，有n个在这个月有消费记录。

  2. 会员分析 - 留存和流失 - 回购率:

    示例：本月100个活跃客户中，有n个是老客户。
    老客户定义：本月前消费过的客户 + 本月在不同日期消费过两次以上的客户。

  3. 会员分析 - 留存和流失 - 30天流失率:

    示例：上个月曾消费过的100个客户中，n个在本月没有消费记录。
    注：上个月：30-60天；本月：30天内。

  4. 会员分析 - 留存和流失 - 60天流失率:

    示例：上上月曾消费过的100个客户中，n个在上月没有消费，且本月也没激活。
    注：上上月：60-90天；上月：30-60天；本月：30天内。

  5. 用户分析 - RFM模型

    见最下面解释。

遗留问题：

  • 升舱和加钟数据分析和展示需根据运营需要做调整。
  • 空间各区利用率，日内和周内利用率数据需根据运营需要做调整。

用户价值RFM模型

模型说明：

指标目的：综合衡量用户价值，未来可用于细化用户管理，活动促销等

用户价值分析的时间跨度为过去半年（180天）。

  1. Recency（最近一次消费指数）- 用户上一次消费间隔天数

    指标目的：越近消费的用户价值越高；但间隔3天和5天的差距不应太大
    R_score = e^(-λ * Recency)； λ = 0.015
    指标区间：0-1，并拉伸至0-100
    指标含义：间隔1天用户，打分接近1；间隔60天的，打分约为0.5

  2. Frequency （消费频次指数）- 时间区间内消费次数

    指标目的：次数0到1代表消费和不消费；1到2是无回购和有回购；次数越多，用户价值衰减越快
    F_score = log (Freqency +1)
    指标区间：0-1，并拉伸至0-100
    指标含义：没消费打分是0，消费1次是0.30，2次 0.47

  3. Monetary （消费力指数）- 时间区间内消费总金额

    指标目的：用户消费金额越高，用户价值边际衰减越快
    M_score = log (Monetary +1)
    指标区间：0-1，并拉伸至0-100
    指标含义：没消费打分是0，消费30元约1.5，100元约2.0，1000元约3.0

  4. 用户价值 - 综合RFM分数

    公式：用户价值 = R_score * (F_score * weight + M_score * (1-weight))
    将所有指标按各自数值分布拉伸到0-100（即最小值为0，最大值为100）
    含义：强化用户留存的重要性。场景用户weight为0.4，强调金额；餐饮用户weight为0.6，强调频次

指标改进：

  • 备选公式：三个数值直接加权相加。
  • 调参：基于实际运营和业务需要调整；例如调整λ，改变R_score衰减速度；或基于业务性质，调整各分数占比。
